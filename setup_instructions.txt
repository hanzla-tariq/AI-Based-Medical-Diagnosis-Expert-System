===============================================
  ShifaBot - AI Medical Diagnosis Expert System
  Setup Instructions
===============================================

PREREQUISITES:
--------------
1. Python 3.10 or higher
2. MySQL Server 8.0 or higher
3. pip (Python package manager)


STEP 1: Create MySQL Database
------------------------------
Open MySQL terminal or phpMyAdmin and run:

    CREATE DATABASE shifabot_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

Or run the schema file directly:

    mysql -u root < backend/database/schema.sql

Note: This project is configured for NO MySQL password.
If your MySQL has a password, edit backend/.env and set DB_PASSWORD=yourpassword


STEP 2: Get a Groq API Key
---------------------------
1. Go to https://console.groq.com/keys
2. Create an account and generate a new API key
3. Copy the key

Then edit backend/.env and replace the value:
    GROQ_API_KEY=your-groq-api-key-here


STEP 3: Install Python Dependencies
------------------------------------
Open a terminal and run:

    cd "e:\Medical Diagnosis System\backend"
    pip install -r requirements.txt


STEP 4: Run the Application
----------------------------
From the backend folder, run:

    python run.py

The application will start at:
    http://localhost:5000

Database tables are created automatically on first run.
The admin account is also seeded automatically.


STEP 5: Access the Application
-------------------------------
Open your browser and navigate to:
    http://localhost:5000

REGULAR USERS:
  1. Click "Get Started" to create an account
  2. Login with your credentials
  3. Start a diagnosis using the Form or Chatbot
  4. View reports and consultation history

      consultation.py      - Consultation model (diagnosis records)

  database/

  requirements.txt         - Python dependencies
  run.py                   - Application entry point

    admin/                 - Admin panel pages (dashboard, users, consultations, reports)


TROUBLESHOOTING:
----------------
- "Access denied for user": Check MySQL credentials in .env
- "ModuleNotFoundError": Run pip install -r requirements.txt again
- "GROQ_API_KEY is not configured": Add your Groq API key to .env
- Port 5000 in use: Change the port in run.py (e.g., port=5001)
- PDF issues: xhtml2pdf is pure Python, no extra system dependencies needed
- Admin can't login: Make sure the server started at least once to seed the account
